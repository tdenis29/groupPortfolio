The Project

This project is the last project in the Odin Projects Foundations curriculum. The goal was to make a basic calculator. It had to include rounded decimal places, the ability to do math with floating point numbers, add a backspace and allow the user to string together operands and operators. For example 12 + 7 - 5 * 3 = 42. We were told specifically NOT to use eval(). My first attempt anded up making an expression string like a graphing calculator which caused a huge increase in difficulty. And an introduction the famous Shunting Yard Algorithm when I tried to make my own eval() function.

Getting the numbers to show up was pretty simple using text-content. The difficult part was going through the logic and deciding when to set variables. Basically I ended up counting clicks. For example we can assume the first click on a calculator is a number and will be the first operand in an expression. After an operator(eg. +, *) is clicked we can ssume the user had the first term they want evaluate so we can set that to a varablie for use later

// first click
if(key.classList.contains('number') && operand1 === null){
handleNumber(keyContent)
updateDisplay()
}

The Above code passes the keyContent(textContent) of the clicked number key to handleNumber() function which controls what gets wrtitten to the screen by setting displayValue with (keyContent)

The Summary

This was a challenge. As I mentioned above I went down the eval() rabbit-hole. I learned about how CPU's do math, reverse Polish notation and most importantly rely on functional programming techniques rather than synchronous. My first version didn't outsource updating the display which led a alot of problems the longer the expression got and ended up with a really long and ugly Event Listener.
